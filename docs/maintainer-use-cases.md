# Maintainer Use Cases

Maintainer Flowboard is intended to support small, practical maintainer workflows.

## Weekly triage planning

A maintainer can load issue and pull request data, review automatic grouping, move cards manually, add private notes, and generate a weekly Markdown summary.

## Information-needed review

Issues that appear to lack reproduction steps, environment details, expected behavior, or actual behavior can be grouped into a queue before the maintainer decides what to ask.

## Review queue clarity

Open non-draft pull requests can be grouped as review candidates. The tool does not decide whether they are good; it only makes the review queue easier to scan.

## Documentation pass

Documentation-related issues and PRs can be gathered into one column so they can be reviewed together.

## Release preparation

Release notes candidates can be grouped before a release, especially items labeled with changelog, release, breaking, or migration.

## Stale review

Older inactive items can be grouped as stale candidates. Maintainer Flowboard does not close them. It only makes them visible for human review.

## New contributor support

Potentially small documentation or sample-data improvements can be easier to spot when the board separates docs candidates from broader work.
