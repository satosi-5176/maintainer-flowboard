# Non-goals

Maintainer Flowboard does not:

- close issues
- merge pull requests
- post replies
- decide project policy
- judge contributor intent
- replace repository-specific workflows
- claim that a classification is correct

It only organizes information before maintainers make decisions.

## Why this matters

The goal is not to automate maintainer authority. The goal is to reduce triage friction by making scattered information easier to review.

Maintainers remain the decision makers.
